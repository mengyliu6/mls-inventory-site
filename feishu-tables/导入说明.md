# 飞书多维表导入说明

这些 CSV 文件可以直接导入飞书多维表。建议在一个多维表应用中分别创建 8 张数据表，表名和文件名保持一致。

## 导入顺序

1. `rooms.csv`
2. `racks.csv`
3. `products.csv`
4. `users.csv`
5. `user_rack_permissions.csv`
6. `sales_people.csv`
7. `field_options.csv`
8. `movements.csv`

## 字段类型建议

- `id`、`room_id`、`rack_id`、`product_id`、`user_id`：文本
- `name`、`vehicle`、`model`、`note`：文本或多行文本
- `role`、`type`、`field`、`source`：单选
- `active`、`can_edit`：复选框
- `x`、`y`、`w`、`h`、`slot_count`、`slot_no`、`stock`、`qty`、`stock_before`、`stock_after`、`sort_order`：数字
- `created_at`、`updated_at`、`at`：日期时间
- `image`：URL 或附件

## 注意

- 字段名请尽量保持英文，不要改成中文字段名，否则 API 需要同步改字段映射。
- `products` 表主要通过 `rack_id + slot_no` 定位货位，`slot_label` 只是方便人工查看。
- `movements` 表每次出入库都新增一条记录，不能只修改 `products.stock`。
- 出库记录的 `sales_person` 建议必填。
